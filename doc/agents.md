# Agents Guide

## 1. 目的
本文件是 `copytrade_v5_muti` 后续 CLOB V2 迁移工作的统一协作规范，适用于 AI agent 与人工工程师。

目标：
- 保证所有协作者基于同一份事实、同一份设计和同一套工作流推进。
- 避免一次性爆改、绕过测试、忽略 cutover 风险或偏离 pUSD / Builder 边界。

## 2. 必须遵守的总原则
- 先阅读 `doc/README.md` 和本目录其余文档，再开始任何代码修改。
- 先更新文档再做大改，文档和代码不一致时先修文档。
- 所有实现以 CLOB V2 为目标，不再为 V1 持续维护兼容路径。
- 不允许一次性重写整个系统；必须按阶段落地。
- 不允许绕过统一交易网关直接在业务代码里依赖 Polymarket SDK。

## 3. 事实来源优先级
1. 官方文档正文
2. 官方 SDK 仓库与示例
3. 当前项目代码与测试
4. 运行日志和本地观测

如不同来源冲突：
- 先信官方正文页面。
- 若官方摘要与正文冲突，必须在文档里记录冲突并以正文为准。
- 若 SDK 实际行为与文档不一致，必须以可复现实验结果修正文档。

## 4. 本项目的固定边界
- 当前主入口是 `copytrade_run.py`。
- 交易执行核心是 `ct_exec.py`。
- 市场解析核心是 `ct_resolver.py`。
- 状态兼容入口是 `ct_state.py`。
- 运行入口还包括 `bounded_copytrade_runner.py`、`persistent_copytrade_runner.py` 和 `windows/*.ps1|*.bat`。

## 5. 未来代码结构要求
- 统一新增 `ct_clob_gateway.py` 作为唯一 SDK 接入层。
- `copytrade_run.py` 不得直接导入 `py_clob_client_v2`。
- `ct_exec.py` 不得直接构造 V2 SDK 订单类型。
- `ct_resolver.py` 继续负责市场可交易状态，不负责签单费用逻辑。
- `ct_state.py` 必须保持对旧状态文件的兼容读取。

## 6. 明确禁止事项
- 禁止保留或新增手工 `fee_rate_bps` 注入逻辑。
- 禁止继续依赖 `/fee-rate` 作为下单签名前置步骤。
- 禁止重新引入旧 Builder HMAC 方案。
- 禁止把自动 `wrap()` / `approve()` 直接塞进主跟单程序。
- 禁止在未完成回归测试前直接宣告迁移完成。
- 禁止在未更新运行器和 Windows 脚本的情况下只改主程序。

## 7. 实施流程要求

### 7.1 开工前
- 先核验官方迁移文档正文日期与 Contracts 页面。
- 记录当前测试基线。
- 确认本目录不是 Git 仓库根目录，并说明采用何种版本化方式。

### 7.2 开发中
- 每次只处理一个阶段性目标。
- 改动接入层时，优先补或改测试。
- 保持业务动作结构稳定，不要顺手重构策略逻辑。
- 对外部协议差异采用网关收口，不要向上层扩散。

### 7.3 开发后
- 必须跑测试。
- 必须更新相关文档。
- 必须说明风险、未覆盖项和下一阶段动作。

## 8. 测试纪律
- 任何接入层改动都必须新增或更新单元测试。
- 任何状态结构改动都必须验证旧状态兼容性。
- 任何 cutover 相关逻辑都必须可演练。
- 保留“现有测试可复跑”作为最低要求。

## 9. pUSD 规则
- 本轮只允许做“人工准备资金 + 程序前置校验”。
- 程序侧可以增强错误提示，但不能在主程序里主动发链上交易。
- 若后续要做自动 `wrap()`，必须新开范围评审，不得夹带进入本轮迁移。

## 10. Builder 规则
- 默认关闭。
- 若未来启用，只能使用 `builderCode`。
- 不得恢复 `POLY_BUILDER_API_KEY`、`POLY_BUILDER_SECRET`、`POLY_BUILDER_PASSPHRASE`、`POLY_BUILDER_SIGNATURE`。

## 11. Cutover 纪律
- 正式切换前必须做预发布联调。
- 正式切换前必须有 runbook。
- cutover 后不能回到 V1 继续运行。
- 遇到挂单状态不一致时，以远端状态为准，不做人肉拼接。

## 12. 文档维护要求
- 所有日期写绝对日期和时区，不写模糊的“今天/明天”。
- 文档中若引用外部事实，尽量附上官方链接。
- 若实施中发现设计需要调整，先改 `doc/` 再改代码。

## 13. 完成定义
只有同时满足以下条件，才能宣称迁移完成：
- 代码已切换到 CLOB V2 接入层
- V1 fee 逻辑已删除
- pUSD 前置校验已到位
- 运行器和 Windows 脚本已同步更新
- 测试通过
- 联调通过
- cutover runbook 已演练

## 14. 结语
本项目的成功标准不是“能在本机跑起来”，而是“可以被团队稳定实施、验证、切换和维护”。所有协作者都应把可验证性、可回顾性和 cutover 安全性放在实现速度之前。
